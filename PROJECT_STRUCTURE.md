# Структура проекта

Этот репозиторий является шаблоном для проектов по аналитике данных.

Структура ориентирована на реальные проекты, которые удобно читать рекрутерам, аналитикам и работодателям.

---

# Общая структура

```

│
├── README.md
├── .gitignore
├── requirements.txt
├── LICENSE
├── .env.example
│
```

# Описание папок

## README.md

Главная страница проекта.

Содержит:

- описание бизнес-задачи;
- используемый стек;
- основные выводы;
- рекомендации;
- ссылки на дашборды;
- результаты проекта.

---

## data/

Хранение данных.

### raw/

Исходные данные.

Никогда не изменяются.

Пример:

```
orders.csv
customers.csv
products.csv
```

---

### processed/

Очищенные данные после обработки.

Например:

```

customers_clean.csv
```

---

### external/

Внешние справочники.

Например:

- словари;
- курсы валют;
- дополнительные таблицы.

# Общий рабочий процесс

1. Поместить исходные данные в `data/raw/`.

2. Проверить качество данных.

3. Выполнить SQL-анализ.

4. Провести EDA в Python.

5. Построить визуализации.

6. Создать дашборд.

7. Подготовить итоговый отчёт.

8. Обновить README проекта.

---

# Принципы организации

- исходные данные никогда не изменяются;
- очищенные данные хранятся отдельно;
- SQL отделён от Python;
- результаты анализа сохраняются отдельно;
- README содержит только итоговую информацию;
- ноутбуки используются для исследования, а не как место хранения финальных результатов.
